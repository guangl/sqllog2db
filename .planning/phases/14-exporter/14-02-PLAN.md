    实现逻辑（按 14-RESEARCH.md Pattern 2 + 14-PATTERNS.md sqlite 一节）：

    1. **Pitfall 1 借用规则**：先读取 `let overwrite = self.overwrite;` 局部变量；然后 `let conn = self.conn.as_ref().ok_or_else(|| Self::db_err("write_template_stats: not initialized"))?;`。

    2. **D-07/D-08 DDL 分支**：若 `overwrite` 为真，执行 `conn.execute("DROP TABLE IF EXISTS sql_templates", [])`，错误用 `Self::db_err(format!("drop sql_templates failed: {e}"))` 包装。

    3. **建表**：始终执行 `conn.execute("CREATE TABLE IF NOT EXISTS sql_templates (template_key TEXT NOT NULL PRIMARY KEY, count INTEGER NOT NULL, avg_us INTEGER NOT NULL, min_us INTEGER NOT NULL, max_us INTEGER NOT NULL, p50_us INTEGER NOT NULL, p95_us INTEGER NOT NULL, p99_us INTEGER NOT NULL, first_seen TEXT NOT NULL, last_seen TEXT NOT NULL)", [])`（DDL 是固定字面量，10 列顺序与 TemplateStats 字段顺序对应）。错误用 `Self::db_err(format!("create sql_templates failed: {e}"))` 包装。

    4. **批量 INSERT 事务**：`conn.execute_batch("BEGIN;")` → 循环 `for s in stats { conn.execute("INSERT INTO sql_templates VALUES (?,?,?,?,?,?,?,?,?,?)", rusqlite::params![s.template_key, s.count, s.avg_us, s.min_us, s.max_us, s.p50_us, s.p95_us, s.p99_us, s.first_seen, s.last_seen])?; }` → `conn.execute_batch("COMMIT;")`。每个 SQL 调用的错误均用 `Self::db_err(format!("...failed: {e}"))` 包装并 `?` 上传。

    5. **日志**：方法末尾 `info!("sql_templates: {} rows written to {}", stats.len(), self.database_url);` 与现有 `finalize` 的 info! 风格一致。

    6. **返回值**：`Ok(())`。任何中间步骤 Err 直接 `?` 向上传播（D-04）。

    7. **不修改 `fn finalize()`**：确认 finalize 仍只执行 COMMIT，不 take/drop self.conn（D-06 前提）。
