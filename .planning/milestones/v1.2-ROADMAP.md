# Milestone v1.2: 质量强化 & 性能深化

**Status:** ✅ SHIPPED 2026-05-15
**Phases:** 7–11
**Total Plans:** 13

## Overview

消灭已知技术债（SQLite 错误可观测性 + SQL 注入防护 + Nyquist 审计补签），补全过滤功能缺口（排除过滤器 FILTER-03），进一步提升 CLI 启动性能（双重 regex 编译消除），并对热路径进行数据驱动的门控审查。

## Phases

### Phase 7: 技术债修复

**Goal**: SQLite 导出错误路径可观测、table_name 配置安全可靠
**Depends on**: Nothing (no code dependencies)
**Requirements**: DEBT-01, DEBT-02
**Plans**: 1 plan

Plans:
- [x] 07-01: DEBT-01 SQLite 静默错误显式化 + DEBT-02 table_name ASCII 校验与 DDL 双引号转义

**Details:**
- `SqliteExporter::validate()` 增加 ASCII 标识符白名单校验（^[a-zA-Z_][a-zA-Z0-9_]*$），非法 table_name 在 `cargo run -- validate` 阶段报错
- `handle_delete_clear_result()` helper 区分"no such table"软失败（忽略）与真实错误（log::warn!），消除静默 `let _ =` 模式
- 5 处 DDL/DML format! 全部改为双引号转义（DROP/DELETE/CREATE/INSERT×3）
- 11 个新测试（config.rs 8 个 + sqlite.rs 3 个），共 673 测试通过

### Phase 8: 排除过滤器

**Goal**: 用户可通过配置"匹配则丢弃"规则精确排除不需要的记录，与现有包含过滤形成完整的 AND/OR-veto 语义
**Depends on**: Phase 7
**Requirements**: FILTER-03
**Plans**: 2 plans

Plans:
- [x] 08-01: filters.rs 核心：CompiledMetaFilters 扩展 exclude 字段、OR-veto should_keep、has_any_filters、validate 方法、21 个单元测试
- [x] 08-02: run.rs has_any_filters() 预计算 + init.rs 配置模板 exclude_* 注释

**Details:**
- `CompiledMetaFilters` 扩展 7 个 `exclude_*` 字段（username/client_ip/sess_id/thrd_id/statement/appname/tag）
- `exclude_veto()` + `include_and()` 拆分为私有方法，should_keep() 主体 ≤ 40 行
- OR-veto 语义：任一 exclude 字段命中即丢弃（短路），include 字段全部命中才保留
- 空 exclude 配置下 pipeline.is_empty() 快路径不受影响
- 21 个新测试覆盖 veto 逻辑与边界情况
