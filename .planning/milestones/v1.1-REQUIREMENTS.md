## v1.1 Requirements (Archived)

### Profiling & Measurement

- [x] **PERF-01**: 开发者能够通过 criterion benchmark 和 flamegraph 定位 CSV 和 SQLite 导出的热路径瓶颈，并生成可复现的基准报告

### CSV 性能

- [x] **PERF-02**: CSV 导出吞吐在 real 1.1GB 日志文件上相比 v1.0 基准（~1.55M records/sec）有可量化提升（目标 ≥10%）
- [x] **PERF-03**: CSV 格式化/序列化路径优化（减少字符串分配、改进 buffer 策略或利用更快的格式化 API）

### SQLite 性能

- [x] **PERF-04**: SQLite 导出使用批量事务（batch INSERT 或显式 transaction 分组），减少单行提交开销
- [x] **PERF-05**: ~~SQLite 导出启用 WAL 模式~~ — 用户决策移除：数据无需崩溃保护，保留高性能模式（JOURNAL_MODE=OFF）
- [x] **PERF-06**: SQLite prepared statement 复用——避免每行重新编译 SQL

### 解析库新特性

- [x] **PERF-07**: 调研 dm-database-parser-sqllog 1.0.0 新 API，若存在零拷贝或批量解析接口则集成到主路径

### 内存/CPU

- [x] **PERF-08**: 热循环内减少堆分配（SmallVec / compact_str 等已有 crate 的充分利用，或消除隐藏 clone）
- [x] **PERF-09**: 所有优化完成后，现有 629+ 测试套件全部通过，无功能退化
