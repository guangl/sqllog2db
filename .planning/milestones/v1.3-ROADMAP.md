# Milestone v1.3: SQL 模板分析 & 可视化

**Status:** ✅ SHIPPED 2026-05-17
**Phases:** 12–16
**Total Plans:** 19

## Overview

v1.3 为 sqllog2db 引入完整的 SQL 模板分析与 SVG 可视化能力。用户可通过 config 启用模板归一化引擎，流式统计所有 SQL 模板的执行频率与耗时分布，并自动输出统计数据（SQLite 表或 CSV 伴随文件）以及四类 SVG 图表。全程恒定内存（hdrhistogram ~24KB/模板），热循环快路径（`pipeline.is_empty()`）完全不受影响。

## Phases

### Phase 12: SQL 模板归一化引擎

**Goal**: 用户可通过 config 启用 SQL 模板归一化，`normalize_template()` 对 sql_text 执行四项变换并生成稳定的模板 key
**Depends on**: Nothing (first v1.3 phase)
**Requirements**: TMPL-01
**Plans**: 3 plans

Plans:
- [x] 12-01-PLAN.md — 在 src/features/sql_fingerprint.rs 实现 normalize_template() 共享扫描引擎（ScanMode enum + 四项变换 + 8 项测试）
- [x] 12-02-PLAN.md — 新增 TemplateAnalysisConfig + FeaturesConfig 接入 + pub use 导出 + init 命令 TOML 模板段
- [x] 12-03-PLAN.md — 在 cli/run.rs 热循环接入 do_template 守卫 + process_log_file/process_csv_parallel 签名透传

**Details:**
- `normalize_template()` 实现为 `scan_sql_bytes(sql, ScanMode::Normalize)` 薄包装，共享 `fingerprint()` 的扫描基础设施
- 四项变换：注释去除（行注释 `--` 和块注释 `/**/`）、IN 列表折叠（统一为 `IN (?)`）、关键字大写、多余空白折叠
- 字符串字面量内的注释符号受保护，不被误判
- 8 项 normalize 测试 + 9 项原有 fingerprint 测试零回归

### Phase 13: TemplateAggregator 流式统计累积器

**Goal**: 用户可启用模板统计聚合，run 结束后每个模板输出 count + avg/min/max + p50/p95/p99 + first_seen/last_seen，热循环零开销快路径完全不受影响
**Depends on**: Phase 12 (normalize_template())
**Requirements**: TMPL-02
**Plans**: 2 plans

Plans:
- [x] 13-01-PLAN.md — 新建 src/features/template_aggregator.rs（TemplateEntry + TemplateAggregator + TemplateStats + hdrhistogram 依赖）
- [x] 13-02-PLAN.md — src/cli/run.rs 侧路径接入：process_log_file 签名替换 + 热循环 observe 调用 + process_csv_parallel map-reduce

**Details:**
- `TemplateAggregator` 不实现 `LogProcessor` trait（避免破坏 `pipeline.is_empty()` 快路径）
- 通过 `Option<&mut TemplateAggregator>` 侧路径参数接入 `process_log_file()`
- 并行 CSV 路径：每个 rayon task 持有独立 `TemplateAggregator`，主线程通过 `merge()` 合并
- hdrhistogram `Histogram<u64>` ~24KB/模板，5M 记录规模下内存不随记录数线性增长

## Milestone Summary

**Key Decisions:**

| 决策 | 理由 |
|------|------|
| TemplateAggregator 不实现 LogProcessor trait | `process()` 接收 `&self`，累积需 `&mut self`；加入 Pipeline 破坏 `pipeline.is_empty()` 快路径 |
| hdrhistogram Histogram<u64> 存储耗时样本 | Vec<u64> 在 5M 记录规模单热模板达 40MB；hdrhistogram ~24KB/模板，误差 <2% |
| plotters SVG-only 配置 | 无字体/图像系统依赖；排除 charts-rs（字体依赖）和 charming（JS 渲染器） |
| observe() 接收已归一化 key（非原始 SQL） | 避免 TemplateAggregator 内部重复归一化，key 稳定性由 Phase 12 保证 |
| 并行 CSV 路径采用 map-reduce merge() 策略 | 每 rayon task 独立 TemplateAggregator，主线程合并，消除锁竞争 |

**Issues Resolved:**

- `normalize_template()` 字面量内注释符号误判（字面量保护机制）
- 并行路径 `process_csv_parallel` observe() 调用点 user 参数同步更新
- user_pie.rs Polygon 扇区渲染与 SVG 输出的 `flush()` 显式调用

**Issues Deferred:**

- VERIFICATION.md 文档补全（Phases 12、13、14、16 — v1.4 tech debt）
- Nyquist VALIDATION.md 合规性（Phases 13/14 nyquist_compliant: false — v1.4 tech debt）
- TMPL-03/03b：独立 JSON/CSV 报告输出（延后至 v1.4+）

**Technical Debt Incurred:**

- Phases 12/13/14/16 缺少 VERIFICATION.md（文档记录缺口，功能已实现）
- 并行路径 write_companion_rows 绕过 ExporterManager（WARNING-01，功能等价，未来扩展需同步）
- user_pie.rs 多处 `#[allow(clippy::cast_*)]`（数值转换业务安全）

**Known Gaps at Close:**

- VERIFICATION.md 缺失（Phases 12、13、14、16）— 文档差距，非功能差距
- Phase 15 VERIFICATION.md 仅覆盖 Wave 1 — Wave 2/3 SVG 渲染层无正式验证文档
- Phases 13/14 Nyquist VALIDATION.md `nyquist_compliant: false`
