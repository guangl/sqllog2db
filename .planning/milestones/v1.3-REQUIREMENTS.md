## v1.3 Requirements (All Completed)

### SQL 模板分析（TMPL）

- [x] **TMPL-01**: 用户可在 config 中启用模板归一化，`normalize_template()` 在 `replace_parameters` 之后对 sql_text 执行注释去除、IN 列表折叠、关键字大小写统一，生成稳定的模板 key
  - **Outcome:** VALIDATED — Phase 12 实现完整，3/3 SUMMARY.md 存在，代码接入热循环 run.rs:239
- [x] **TMPL-02**: 用户可启用模板统计聚合，run 结束后每个模板输出 count + avg/min/max + p50/p95/p99 + first_seen/last_seen；使用 hdrhistogram（~24 KB/模板），禁止 Vec 全量样本存储
  - **Outcome:** VALIDATED — Phase 13 实现完整，hdrhistogram ~24KB/模板
- [x] **TMPL-04**: SQLite 导出时自动写入 `sql_templates` 统计表；CSV 导出时自动生成 `*_templates.csv` 伴随文件
  - **Outcome:** VALIDATED — Phase 14 实现完整，两路径均有集成测试

### SVG 图表生成（CHART）

- [x] **CHART-01**: 用户可在 config 的 `[features.charts]` 中指定 `output_dir`，启用后 run 结束时自动生成 SVG 文件到该目录
  - **Outcome:** VALIDATED — Phase 15 实现，ChartsConfig + validate() 依赖检查
- [x] **CHART-02**: 生成 Top N 模板执行频率横向条形图（SVG），N 可在 config 中配置
  - **Outcome:** VALIDATED — Phase 15，`top_n_frequency.svg` draw_frequency_bar 实现
- [x] **CHART-03**: 生成全局耗时分布直方图（SVG），使用 hdrhistogram bucket 数据
  - **Outcome:** VALIDATED — Phase 15，`latency_histogram.svg` draw_latency_hist 对数轴实现
- [x] **CHART-04**: 生成 SQL 执行频率时间趋势折线图（SVG），小时粒度
  - **Outcome:** VALIDATED — Phase 16，`frequency_trend.svg` draw_trend_line BTreeMap hour_counts
- [x] **CHART-05**: 生成用户 / Schema 执行占比饼图（SVG）
  - **Outcome:** VALIDATED — Phase 16，`user_schema_pie.svg` draw_user_pie HSL 颜色生成

## Deferred Requirements

- **TMPL-03**: 用户可指定独立 JSON 报告输出路径，run 结束后生成包含 p50/p95/p99/histogram_buckets 的机器可读报告
  - **Outcome:** DEFERRED — 推迟至 v1.4+
- **TMPL-03b**: 用户可指定独立 CSV 报告输出路径，run 结束后生成 DBA 可直接用 Excel 打开的模板统计摘要
  - **Outcome:** DEFERRED — 推迟至 v1.4+

## Out of Scope

| Feature | Reason |
|---------|--------|
| 交互式 HTML 仪表盘 | 需要 JS 运行时，破坏静态文件约束 |
| 精确 p50/p95/p99（Vec 全量存储） | 内存不可控；hdrhistogram 误差 <2% 对 DBA 已足够 |
| 模板相似度聚类 | O(N²) 成本，超出当前里程碑范围 |
| OR 条件组合（FILTER-04） | 继承自 v1.1 Out of Scope 决策 |
| 跨字段联合条件（FILTER-05） | 继承自 v1.1 Out of Scope 决策 |
