# Plan 16-05: 接入新图表模块

## Goal

在 `src/charts/mod.rs` 中声明两个新模块，更新 `generate_charts()` 函数，
按 `cfg.trend_line` / `cfg.user_pie` 开关分别调用折线图和饼图渲染函数。

## Dependency

依赖 Plan 16-03（`trend_line.rs` 存在）和 Plan 16-04（`user_pie.rs` 存在）。

## Tasks

### Task 1: 声明新模块

在 `src/charts/mod.rs` 顶部的 `pub mod` 声明后追加：

```rust
pub mod trend_line;
pub mod user_pie;
```

### Task 2: 更新 generate_charts() 函数

当前函数只收集 `entries: Vec<ChartEntry<'_>>`，Phase 16 需要额外数据。
更新函数签名以使用 `TemplateAggregator` 引用（已有）：

在函数体内，按开关调用新图表：

```rust
if cfg.trend_line {
    let hour_data: Vec<(&str, u64)> = agg.iter_hour_counts().collect();
    let path = output_dir.join("frequency_trend.svg");
    trend_line::draw_trend_line(&hour_data, &path)?;
}

if cfg.user_pie {
    let user_data: Vec<(&str, u64)> = agg.iter_user_counts().collect();
    let path = output_dir.join("user_schema_pie.svg");
    user_pie::draw_user_pie(&user_data, cfg.top_n, &path)?;
}
```

注意：`iter_hour_counts()` 返回 `impl Iterator<Item = (&str, u64)>`，
但返回的 `&str` 生命周期绑定到 `agg`，collect 到 `Vec<(&str, u64)>` 后可传给渲染函数。

### Task 3: 更新 generate_charts() 中的 entries 收集

现有代码：
```rust

```

此行仍保留，frequency_bar 和 latency_hist 仍使用 entries。
新增的 hour_data / user_data 独立收集。
